# 音频转码插件 (Audio Transcoder Plugin)

## 📝 插件简介

自动检测新整理的视频文件，对 EAC3/AC3 单音轨进行转码为 AAC 立体声外挂音轨。

本插件专为 MoviePilot V2 设计，能够智能监控视频目录，自动处理多声道音频格式，生成符合 Emby/Jellyfin/Plex 等媒体服务器规范的外挂音轨文件。

## ✨ 主要功能

### 核心特性

1. **智能监控** - 自动监控视频整理后的目录（支持多个目录配置）
2. **音频检测** - 使用 ffprobe 精确检测视频文件的音轨信息
   - 音轨数量分析
   - 音频编码格式识别（EAC3、AC3、AAC 等）
   - 声道数检测
3. **智能过滤** - 自动跳过不需要处理的文件
   - 多音轨视频（>1条音轨）→ 跳过
   - 单音轨非 EAC3/AC3 格式 → 跳过
   - 已有外挂音轨的视频 → 跳过
4. **音频转码** - 将 EAC3/AC3 单音轨提取并转码为 AAC 立体声
5. **规范命名** - 按 Emby 规则命名外挂音轨文件
   - 格式：`<视频文件名>.<语言代码>.aac`
   - 示例：`Movie.2023.chi.aac`

### 安全保证

- ✅ **不修改原视频文件** - 保持原始文件完整性
- ✅ **不修改字幕文件** - 字幕文件不受影响
- ✅ **不覆盖已有音轨** - 检测到已有外挂音轨则跳过
- ✅ **只创建新的 .aac 外挂音轨文件** - 安全的追加操作

## 🔧 技术实现

### 工作流程

```
视频文件整理完成
    ↓
触发事件监听 / 定时扫描
    ↓
检测是否为视频文件 (.mp4/.mkv/.avi 等)
    ↓
使用 ffprobe 分析音轨信息
    ↓
判断条件：
├─ 音轨数 > 1 → 跳过
├─ 音轨数 = 1 且格式为 EAC3/AC3 → 继续
└─ 其他情况 → 跳过
    ↓
检查是否已有外挂音轨
├─ 已存在 → 跳过
└─ 不存在 → 开始转码
    ↓
提取音频并转码为 AAC 立体声
    ↓
按 Emby 规范命名并保存
    ↓
完成
```

### 依赖工具

- **ffmpeg** - 音频提取和转码
- **ffprobe** - 音视频流信息分析

> ⚠️ 请确保系统已安装 ffmpeg 和 ffprobe，并可在命令行中直接调用。

## 📋 配置说明

### 基本配置

在插件设置页面可以配置以下选项：

1. **启用插件** - 开启/关闭插件功能
2. **监控目录** - 设置需要监控的视频目录（支持多个目录）
3. **转码模式** - 选择自动模式或手动模式
4. **日志级别** - 调整日志输出详细程度

### 转码参数

插件使用以下默认转码参数：

- **输出格式**: AAC (Advanced Audio Coding)
- **声道数**: 立体声 (2 channels)
- **比特率**: 128 kbps
- **采样率**: 48000 Hz

## 🎯 使用场景

### 适用场景

- 📺 下载的影片只有 EAC3/AC3 多声道音轨
- 🎬 希望在不修改原文件的情况下添加立体声音轨
- 📱 移动设备播放时兼容性更好
- 🔊 某些播放器不支持多声道解码

### 不适用场景

- ❌ 已经是 AAC 或其他兼容格式的视频
- ❌ 包含多条音轨的视频（通常已有立体声备选）
- ❌ 已经有外挂 AAC 音轨的视频

## 🐛 常见问题

### Q: 为什么我的视频没有被处理？

A: 可能的原因：
1. 视频有多条音轨（插件只处理单音轨视频）
2. 音频格式不是 EAC3 或 AC3
3. 已经存在外挂 AAC 音轨
4. 监控目录配置不正确

### Q: 转码后的音轨文件在哪里？

A: 外挂音轨文件与视频文件在同一目录下，命名为 `<视频文件名>.<语言代码>.aac`

### Q: 如何查看转码日志？

A: 在 MoviePilot 的日志管理中筛选插件名称 "音频转码插件" 即可查看详细日志。

### Q: 支持哪些视频格式？

A: 支持所有 ffmpeg 支持的视频容器格式，常见的包括：
- MP4 (.mp4, .m4v)
- Matroska (.mkv)
- AVI (.avi)
- MOV (.mov)
- 以及其他常见格式
